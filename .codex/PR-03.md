# PR-03.md — Validator: filesystem + JSON + CBOR presence (path + find)

## Repo
You are working in repo: **greentic-integration**

## Goal
Implement `greentic-integration-validator` as a small assertion CLI designed to be called from `.gtest` scripts.

Validator must:
- be schema-agnostic (no Greentic types required)
- support CBOR reading and *presence* checks:
  - path-based existence / equality checks
  - recursive “find” checks (value exists anywhere)
- return consistent exit codes:
  - 0 = pass
  - 2 = validation failed (assertion false)
  - 1 = internal error (IO, parse error, bug)

## CLI Surface

### Filesystem
- `file exists <path>`
- `file not-exists <path>`
- `file contains <path> <substring>`
- `file regex <path> <pattern>`
- `dir contains <dir> <glob>` (optional v1; ok to omit if too much)

### JSON
- `json path <file> <path> --exists`
- `json path <file> <path> --eq <json_literal>`
Path format:
- dot keys: `a.b.c`
- array index: `arr[0].x`
- quoted keys for special chars: `obj["weird-key"]`

### CBOR
Use `serde_cbor::Value` (or `ciborium::value::Value`). Keep schema-agnostic.

#### 1) Path-based
- `cbor path <file> <path> --exists`
- `cbor path <file> <path> --eq <json_literal>`
- `cbor path <file> <path> --type map|array|string|bytes|int|bool|null|float` (optional)

Path rules (same as JSON), plus:
- (Optional but recommended) wildcard array selector: `[*]`
  - Example: `components[*].id --eq "templates"`
Implementation can return multiple matches for wildcard paths.

Semantics:
- `--exists`: pass if at least one match exists
- `--eq`: if multiple matches:
  - default: pass if any equals
  - if `--all`: require all matches equal
  - if `--count N`: require N matches (combine with exists or eq)

#### 2) Recursive find
- `cbor find <file> --key <k>` (search map keys that are strings)
- `cbor find <file> --string <s>` (search any string value equal)
- `cbor find <file> --regex <pattern>` (search any string value matches)
- (optional) `cbor find <file> --bytes-hex <hex>`

## Implementation Details
### Structure
- Use clap subcommands:
  - `File { cmd: FileCmd }`
  - `Json { cmd: JsonCmd }`
  - `Cbor { cmd: CborCmd }`
- Each leaf command returns `Result<bool>` where bool is assertion pass/fail.
- Main maps:
  - Ok(true) => exit 0
  - Ok(false) => exit 2
  - Err(_) => exit 1

### Shared path parser
Implement a small reusable path parser in `core` (or locally in validator):
- Parse a string like `a.b[0].c["x-y"]` into segments:
  - Key("a"), Key("b"), Index(0), Key("c"), Key("x-y")
  - IndexWildcard for `[*]` if you implement it.
Keep it intentionally small—no full JSONPath.

### JSON path evaluation
- Load `serde_json::Value`
- Navigate segments and return matches (`Vec<&Value>` or owned clones).

### CBOR path evaluation
- Load CBOR bytes and parse into `serde_cbor::Value`
- Implement navigation across:
  - maps: look up by string key only (if key is not a string, treat as not found)
  - arrays: by index or wildcard
- Return matches as `Vec<serde_cbor::Value>` (owned) or references; whichever is simpler.

### Comparing `--eq <json_literal>`
- Parse the literal using `serde_json::from_str`.
- Convert CBOR Value into a JSON-ish Value for comparison:
  - Null/bool/int/float/string/array/map map naturally
  - Bytes:
    - for `--eq`, do NOT support bytes equality in v1 unless you define a representation
    - emit a clear error: “bytes compare not supported; use --type bytes or cbor find --bytes-hex”
This keeps behavior explicit.

### CBOR find
DFS traversal:
- if `--key`: check map keys that are strings
- if `--string`: check string values
- if `--regex`: check string values against regex (use `regex` crate)

## Tests

- find behavior
