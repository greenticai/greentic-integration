# Title: E2E: provider-core only for secrets + messaging + events

## Goal
Create end-to-end tests that prevent regressions and guarantee the migration is complete.

## CI contract
- Set `GREENTIC_PROVIDER_CORE_ONLY=1` for all E2E runs.
- Fail if any legacy provider protocols are used (compile-time and runtime).

## Test suite
1. **Secrets**
   - Use an in-memory provider-core secrets provider pack
   - Onboard it (deployer path or fixture install)
   - Run flow:
     - `put key=a value=1`
     - `get key=a` → assert `1`
2. **Messaging**
   - Use dummy provider-core messaging pack
   - Flow: `send` → assert receipt contains message_id and echo text
3. **Events**
   - Use dummy provider-core events pack
   - Flow: `publish topic=t event={...}` → assert receipt and/or stored state
4. **Store indexing**
   - Index a fixture pack with provider extension
   - Assert provider index contains provider_type/capabilities/ops
5. **Schema onboarding**
   - Confirm deployer prompts + JSON schema validation + `validate-config` called

## Fixtures
- Add fixture packs under `tests/fixtures/packs/`:
  - `provider_secrets_dummy.gtpack`
  - `provider_messaging_dummy.gtpack`
  - `provider_events_dummy.gtpack`
- Include schemas under `schemas/**` within each pack fixture.

## Acceptance criteria
- Tests pass deterministically in CI.
- No legacy path usage is possible without tests failing.
