## Goal
Implement the core `.gtest` parsing and `${VAR}` substitution logic in `greentic-integration-core`.

The `.gtest` format is:
- Each non-empty non-comment line is either:
  1) a directive line starting with `@`
  2) a command line (argv-like)

Comments start with `#` and may appear on their own line (do not support inline trailing comments in v1).

## Deliverables
1) Parser that reads a `.gtest` file and produces a `TestPlan` = ordered `Vec<Step>`.
2) Support directives:
- `@set KEY=VALUE`
- `@env KEY=VALUE`
- `@cd PATH`
- `@timeout DURATION` (e.g. `60s`, `500ms`, `2m`)
- `@expect exit=0` or `@expect exit!=0`
- `@capture NAME`
- `@print NAME` (directive; runner implements effect later)
3) Variable substitution:
- `${VAR}` replaced in directive args and command argv tokens.
- Sources (precedence highâ†’low):
  1. Test variables from `@set`
  2. Environment variables set via `@env` (within context)
  3. Process environment variables
  4. Built-ins (provided by runner later): `WORK_DIR`, `TEST_DIR`, `REPO_ROOT`, `TMP_DIR`
- If a variable is missing, fail with a clear error indicating the variable name and line number.

4) Tokenization for command lines:
- Basic shell-like splitting with quotes:
  - Support double quotes `"..."` and single quotes `'...'`
  - Support backslash escapes inside double quotes for `\"` and `\n` (minimal)
- This is not a full shell. No pipes, redirects, globbing, or subshells.
- Return `argv: Vec<String>`.

## Implementation Details
### Model types (in `core::model`)
- `Step { line_no: usize, raw: String, kind: StepKind }`
- `StepKind::Directive(Directive)` or `StepKind::Command(CommandLine)`
- `Directive` enum:
  - `Set { key, value }`
  - `Env { key, value }`
  - `Cd { path }`
  - `Timeout { duration: Duration }`
  - `ExpectExit { equals: Option<i32>, not_equals: Option<i32> }`
  - `Capture { name }`
  - `Print { name }`
- `CommandLine { argv: Vec<String> }`
- `TestPlan { path: PathBuf, steps: Vec<Step> }`

### Parser (in `core::parse`)
- `parse_gtest_file(path: &Path) -> Result<TestPlan>`
- For each line:
  - trim whitespace
  - ignore empty lines
  - ignore lines starting with `#`
  - if starts with `@`: parse directive with strict grammar
  - else: tokenize into argv

Directive grammar (v1):
- `@set KEY=VALUE` (KEY is `[A-Za-z_][A-Za-z0-9_]*`)
- `@env KEY=VALUE` (same KEY rules)
- `@cd PATH` (PATH is the rest of line after `@cd `, trimmed)
- `@timeout DURATION` where DURATION matches `^(\d+)(ms|s|m|h)$`
- `@expect exit=0` or `@expect exit!=0` (int can be any i32)
- `@capture NAME` (NAME same rules as KEY)
- `@print NAME`

### Substitution (in `core::substitute`)
- `SubstitutionContext` holds:
  - `test_vars: HashMap<String,String>`
  - `env_vars: HashMap<String,String>` (from directives, *not* OS env)
  - `builtin: HashMap<String,String>`
- `substitute(s: &str, ctx: &SubstitutionContext, line_no: usize) -> Result<String>`
- Substitute all `${VAR}` occurrences (non-nested).
- Reject `${}` and invalid variable names.
- Provide unit tests for:
  - precedence behavior
  - missing vars
  - multiple substitutions in one string
  - no substitution case

### Errors
- Use `thiserror` with variants including:
  - `ParseError { line_no, message }`
  - `MissingVar { line_no, var }`
  - `TokenizeError { line_no, message }`
